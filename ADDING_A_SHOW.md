## Mental model

VaultVision hosts **no video**. Every show is a list of pointers into
[archive.org](https://archive.org) items, and the player builds a stream URL of
the form:

```
https://archive.org/download/<ITEM_ID>/<PATH/TO/FILE.mp4>
```

There is **no build step and no database**. A show is a folder of plain files
plus one line of HTML. The web player (`engine/viewer.html`), the show screen
(`show.html`) and the guide (`index.html`) all read the same `data.js`; the Roku
channel reads a JSON catalog generated *from* those same files.

Three consumers, one source of truth:

| Consumer | Reads |
|---|---|
| `index.html` (guide) | its own hard-coded card + `localStorage` progress |
| `show.html` (info screen) | `shows/<Id>/data.js` |
| `engine/viewer.html` (player) | `shows/<Id>/data.js`, `shows/<Id>/captions.js` |
| Roku / other clients | `roku/catalog.json` + `roku/shows/<Id>.json`, generated by `VaultVisionRoku/tools/build_catalog.py` from the two above |

---

## The six things a show needs

Pick a **ShowId** first: PascalCase, ASCII letters and digits only, no spaces or
punctuation (`Newhart`, `DickVanDyke`, `Y2KTheMovie`, `AlfredHitchcockPresents`).
Disambiguate remakes with a year (`It1990`, `LandOfTheLost1991`). This id is the
folder name, the `?show=` parameter, the poster filename stem and the
`localStorage` prefix — it must be identical everywhere.

| # | Path | Required | Notes |
|---|---|---|---|
| 1 | `shows/<ShowId>/data.js` | yes | the episode list + modifiers |
| 2 | `shows/<ShowId>/captions.js` | yes | `window.CAPTIONS = {};` if you have none |
| 3 | `shows/<ShowId>/CREDITS.md` | yes | attribution + source notes |
| 4 | `art/<ShowId>.<ext>` | yes | poster, `.jpg`/`.png` preferred |
| 5 | one `<a class="card">` line in `index.html` | yes | under the right `<h2>` genre |
| 6 | `roku/catalog.json` + `roku/shows/<ShowId>.json` | yes | **generated**, never hand-edited |

Nothing else in the repo changes. If a diff touches `engine/viewer.html`,
`show.html` or `tv.js` while adding a show, something has gone wrong — those are
engine files, not content.

## Step 1 — Find and vet the source

### 1a. Locate the item(s)

Search archive.org, then pull the metadata JSON — this is the authoritative file
list, and the only thing you should build the episode list from:

```bash
curl -s "https://archive.org/metadata/<ITEM_ID>" > /tmp/item.json
python3 - <<'EOF'
import json
d = json.load(open('/tmp/item.json'))
for f in d['files']:
    if f['name'].lower().endswith(('.mp4', '.mkv', '.avi', '.ogv', '.webm', '.m4v')):
        print(f.get('source'), f.get('format'), f.get('length'), f['name'])
EOF
```

**Never infer filenames.** Guessed paths 404 silently and the player retries,
then skips to the next episode — which looks like a random ordering bug much
later.

One show may span several items (see `Newhart`, eight per-season items). That is
fully supported: each episode row carries its own item id.

### 1b. Vet the files

| Check | Why | What to do |
|---|---|---|
| `format` is `h.264` / MPEG4 and audio is AAC/MP3 | browsers can't decode AC3/DTS; the video plays with silent audio | prefer the `derivative` `.mp4`; the engine auto-prefers a same-stem `.mp4` for non-web hints, but only if one exists |
| `source: original` `.mp4` with no derivative | Archive skips deriving files uploaded as `.mp4` | probe it (`ffprobe`) — usually already h.264/AAC and fine to stream directly |
| duplicate cuts (with/without commercials) | duplicate rows in the guide | list one; mention the other in `CREDITS.md` |
| bumper/clip files, "part 1 of" fragments | half-length or bogus entries | exclude them |
| truncated rips (`length` far below the rest) | look broken to a viewer | list them but flag with [`incompleteEpisodes`](#incompleteepisodes) |
| filenames numbered 001…N | upload order ≠ broadcast order | see 1c |

### 1c. Verify order against an external source

**Do not trust filenames for season/episode placement.** Check every episode
title against Wikipedia's episode list (or an equivalent) and pin each file to
its real season/episode. Green Acres is the cautionary tale: its files run
`001`–`170` straight through, several Season 1 entries are transposed, and the
sequence skips `027` while still ending at `170` — slicing the range into
seasons mislabels most of the back half of Season 1.

When the upload's spelling differs from the canonical title, put the **canonical
title in the guide** and note the discrepancy in `CREDITS.md`.

## Step 2 — Build the episode list

### The episode row

```js
[itemId, title, fileHint]
```

| Position | Type | Required | Meaning |
|---|---|---|---|
| `0` | string | yes | archive.org item id (from `archive.org/details/<id>`) |
| `1` | string | yes | display title — **parsed**, see grammar below |
| `2` | string | only when the item holds more than one video | exact file path inside the item, including any folders |

Omit `fileHint` only when the item contains exactly one relevant video: the
player then picks the first `.mp4` (else `.ogv`/`.webm`) it finds.

`fileHint` may contain `/` for items with nested folders — the player encodes
each path segment separately, so slashes survive. Copy the path exactly as it
appears in the metadata, including spaces, capitalisation and double spaces.

### Title grammar (this is load-bearing)

Titles are split on `" - "` (space-hyphen-space) and the parts mean:

```
SHOW NAME - S01E01 - Episode Name
    [0]        [1]        [2:]
```

* `parts[1]` → the **episode code**, shown in the code column, matched by
  `incompleteEpisodes`, and scanned for `/S(\d+)/i` to derive the season.
* `parts[2:]` rejoined with `" - "` → the episode name (so names may contain
  `" - "` freely).
* No `S\d+` in the code ⇒ **season 1**. Fine for single-season shows.
* A missing name is legal (`ALFRED HITCHCOCK PRESENTS - S01E01`).
* The show-name portion must not contain `" - "`. Hyphens without spaces are
  fine (`X-Men`, `Beavis and Butt-Head`).

Conventions in use:

| Kind | Title form | Example |
|---|---|---|
| Standard episode | `SHOW - S01E01 - Name` | `NEWHART - S01E01 - In the Beginning` |
| Unaired pilot / special | `S00E00` + [`seasonNames`](#seasonnames) | `THE DICK VAN DYKE SHOW - S00E00 - Head of the Family (Pilot)` |
| Double episode | `S02E10-11` | `BEAVIS AND BUTT-HEAD - S02E10-11 - …` |
| Split cartoon halves | `S03E01a` / `S03E01b` + [`mergeParts`](#mergeparts) | `THE REN & STIMPY SHOW - S03E01a - …` |
| Dated broadcast block | `SHOW - MM/DD/YYYY` | `SNICK - 08/15/1992` |
| Undated block | `SHOW - <descriptor>` | `FOX KIDS - early 1990s (undated)` |
| Movie part | `SHOW - Part 1` | `STEPHEN KING'S IT - Part 1` |

Show names are conventionally upper-cased in the title string. It is only a
convention — the player marquees whatever you write.

### The episode key

Several structures are keyed by an **episode key**, which is:

```
"<itemId>::<fileHint>"     when the row has a fileHint
"<itemId>"                 when it doesn't
```

Used by `durations`, `shows`, `film`, `segmentTimestamps`, `window.CAPTIONS`,
loose `.vtt` filenames, and the watched-episode set. Get one character wrong and
the entry is silently ignored — no error, just a missing runtime or caption
track. **Generate these keys programmatically from the same list you generate
the episodes from.**

### Durations

`durations` maps episode key → runtime in **seconds** (integer). Populate it
whenever the metadata has lengths — it is what shows a runtime in the episode
list, and it is *mandatory* for [`mergeParts`](#mergeparts).

Archive's `length` field is usually a seconds string (`"1568.97"`) but sometimes
`"HH:MM:SS"`. Handle both:

```python
def secs(v):
    if ':' in v:
        p = [float(x) for x in v.split(':')]
        while len(p) < 3:
            p.insert(0, 0.0)
        return int(round(p[0] * 3600 + p[1] * 60 + p[2]))
    return int(round(float(v)))
```

Leaving `durations: {}` is acceptable (22 shows do) — the list just shows no
runtimes.

## Step 3 — Write `shows/<Id>/data.js`

### Canonical skeleton

```js
window.SHOW = {
  id: "Newhart",
  title: "Newhart",
  art: "Newhart.jpg",
  storageProgressKey: "newhart-progress",
  storageSettingsKey: "newhart-settings",
  nestedPaths: true,
  grouping: "season",
  segments: "none",
  episodes: [
  ['NewhartSeries', "NEWHART - S01E01 - In the Beginning",
    "Newhart - 101 - In the Beginning.mp4"],
  // …
],
  durations: {
  "NewhartSeries::Newhart - 101 - In the Beginning.mp4": 1569,
  // …
},
};
```

### Core fields

| Field | Type | Required | Rule |
|---|---|---|---|
| `id` | string | conventional | must equal the folder name. The player uses the URL's `?show=`, so this is documentation — but keep it right |
| `title` | string | **yes** | display name; also the HUD label and the fallback segment name |
| `art` | string | **yes** | filename inside `art/`, with extension |
| `storageProgressKey` | string | **yes** | **must** be `<ShowId>.toLowerCase() + "-progress"` — the guide's Resume row derives this key from the card's `?show=` id and will never find your show otherwise |
| `storageSettingsKey` | string | **yes** | **must** be `<ShowId>.toLowerCase() + "-settings"` |
| `grouping` | `"season"` \| `"flat"` | **yes** | see below |
| `segments` | `"none"` \| `"static"` \| `"film"` \| `"lineup"` | **yes** | see below |
| `episodes` | array | **yes** | the rows from Step 2, in play order |
| `durations` | object | yes (may be `{}`) | episode key → seconds |
| `nestedPaths` | boolean | optional | **no-op**. Nothing reads it; path encoding is unconditional. Present in 83 of 84 shows; keep it for consistency, set `true` when any `fileHint` contains `/` |

### Syntax constraints

`data.js` is loaded as a `<script>` by the browser **and** parsed by a small
hand-written reader (`VaultVisionRoku/tools/jsdata.py`) that supports only:
objects, arrays, strings, numbers, booleans, `null`, `//` and `/* */` comments,
a leading `const NAME = '…';` referenced later, and `+` string concatenation.

So: **no functions, template literals, spread, computed keys, `.map()`, or any
expression.** Plain literals only, or the Roku build fails and the show
disappears from the catalog.

Both quote styles are fine. Trailing commas are fine.

## Modifier reference

Everything below is optional. Defaults are what you get by omitting the field.
"Roku" says whether `build_catalog.py` carries the behaviour to the Roku channel.

---

### `grouping`

**Values:** `"season"` (default in practice) | `"flat"` · **Required field.**

Decides whether the episode list is split into season tabs.

* `"season"` — seasons come from `/S(\d+)/i` in the episode code; codes without
  one land in season 1. Use for any real series.
* `"flat"` — one implicit list, no season tabs. `show.html` also relabels the
  count as "N parts" instead of "N episodes". Use for TV movies, and for
  broadcast-block shows (recordings, not episodes).

**Roku:** yes — `grouping`, and `seasons[]` is prebuilt in the per-show JSON.

---

### `segments`

**Values:** `"none"` | `"static"` | `"film"` | `"lineup"` · **Required field.**

Decides what the REW/FF (prev/next) buttons do, and what the `ch` on-screen
popup names.

| Value | One episode row is… | REW/FF | Extra fields required |
|---|---|---|---|
| `"none"` | one self-contained episode | pages to the previous/next episode, wrapping at the ends | — |
| `"static"` | a multi-show recording whose lineup is known per recording | snaps to the previous/next show *within* the recording | [`shows`](#shows) |
| `"film"` | a feature film (or a tape holding several) | snaps between films on a multi-film tape; on a single film, jumps to start/end | [`film`](#film) |
| `"lineup"` | a dated broadcast block whose lineup is derived from the date | snaps between the four slots | [`lineups`](#lineups) |

Segment boundaries are **evenly divided** across the runtime unless
[`segmentTimestamps`](#segmenttimestamps) supplies real ones (`"lineup"` always
assumes four slots).

`"film"` also changes the episode-list layout: the name column is separated from
the runtime column and long names marquee on hover.

**Roku:** the value is carried, but `shows` / `film` / `lineups` /
`segmentTimestamps` are **not** — segment navigation is web-only today.

---

### `shows`

**Type:** `{ [episodeKey]: string[] }` · **Required when `segments: "static"`.**
Also used by `segments: "film"` for multi-film tapes.

Names the programmes inside each recording, in broadcast order.

```js
shows: {
  'VHSFoxMMPRWOC': ['Mighty Morphin Power Rangers'],
  '20240810_20240810_1448': ['X-Men', 'The Tick', 'Superman', 'The Incredible Hulk'],
},
```

* Under `"static"`, a recording with no entry falls back to `[SHOW.title]` — but
  the map object itself is dereferenced unguarded, so **`shows` must exist**.
* Under `"film"`, an entry means "this file is a tape holding several films";
  no entry means "single film", and the name then comes from `film`.
* Keys are episode keys — for a recording with no `fileHint`, just the item id.

**Roku:** no.

---

### `film`

**Type:** `{ [episodeKey]: string }` · **Required when `segments: "film"`.**

Maps each file to the film's real title, so the list and OSD read
`Big Trouble in Little China` rather than the wrapper show's name.

```js
film: {
  "joe-bob-briggs-monstervision::JoeBobBriggs_Carrie.mp4": "Carrie",
},
```

The list-label path reads `SHOW.film[key]` **without** optional chaining, so
under `segments: "film"` the `film` object must exist even if sparse.

**Roku:** no.

---

### `lineups`

**Type:** `[[isoDate, string[]], …]` · **Required when `segments: "lineup"`.**

A dated table of what the block's four slots were, applied to every recording
airing on or after that date:

```js
lineups: [ // [first air date using this lineup, shows]
  ['1992-08-15', ['Clarissa Explains It All', 'Roundhouse', 'The Ren & Stimpy Show', AYAOTD]],
  ['1994-06-01', ['The Adventures of Pete & Pete', 'Roundhouse', 'The Ren & Stimpy Show', AYAOTD]],
],
```

Hard requirements:

* Sorted **ascending** by date, and the first entry should be on or before the
  earliest episode's date.
* Four entries per lineup (boundaries are `duration / 4`).

Dating an episode title, in the order the player tries them:

1. `MM/DD/YYYY` anywhere in the title — the normal case, and the only one that
   also feeds the airdate display.
2. a bare four-digit year — read as January 1st of it. Good enough to land on
   the right slate most of the time, since lineups change once or twice a
   season. Use this for a tape you can't date precisely; don't invent a
   specific date it doesn't have.
3. neither, or a date earlier than every row in the table — falls back to
   `[SHOW.title]`, so the `ch` popup names the block itself.

A `const NAME = '…'` above `window.SHOW` for a repeated show name is supported by
both the browser and the Roku parser (see `shows/SNICK/data.js`).

**Roku:** no.

---

### `segmentTimestamps`

**Type:** `{ [episodeKey]: [[startSec, endSec, label], …] }` · Optional.

Hand-timed real segment boundaries for one recording. Takes priority over the
evenly-divided guesses for every `segments` mode, and the third element becomes
the `ch` popup's label.

```js
segmentTimestamps: {
  'vhs-tv-fox-kids-monkey-magic-woc': [
    [84, 1633, "Monkey Magic - The Invincible Guardian"],
    [1633, 3317, "Monkey Magic - Sanzoh to the Rescue"],
  ],
},
```

Only `[0]` (start) and `[2]` (label) are read; keep `[1]` accurate anyway as
documentation. Supply it for as many or as few recordings as you have timings
for.

**Roku:** no.

---

### `mergeParts`

**Type:** `boolean` (default `false`) · Optional.

Folds consecutive rows whose code differs only by a trailing letter
(`S03E01a`, `S03E01b`) into **one logical episode**: the parts play back to
back, the scrubber and runtime treat them as a single timeline, and the merged
title becomes `Name A / Name B`.

```js
mergeParts: true, // S03 splits each episode into a/b cartoons — play them as one
```

Requirements:

* Codes must match `^(.*\d)([a-z])$` — `S03E01a`, not `S03E01A` or `S03E01-a`.
* Parts must be **adjacent** in `episodes` and in order.
* `durations` **must** cover every part, or part offsets are 0 and seeking
  across a seam misbehaves.
* Rows without a trailing letter never merge; mixing merged and unmerged rows in
  one show is fine.

Only `RenAndStimpy` uses this.

**Roku:** no — parts appear as separate episodes there.

---

### `seasonNames`

**Type:** `{ [seasonNumber]: string }` · Optional.

Overrides the season heading in the **player's** episode list (both the flat
HUD panel and the 3D canvas list).

```js
seasonNames: { 0: "Unaired Pilot" },
```

Note the deliberate limitation: `show.html`'s season tabs and the Roku channel
always render `Season N`. `seasonNames` is player-only.

---

### `introSkipSeconds`

**Type:** `number` (seconds) · Optional.

Starts playback this far in — for rips with a baked-in station intro before
every episode.

```js
introSkipSeconds: 54,
```

Applies **only** on the first part of an episode and **only** when not resuming
a saved position. Currently used by `AlfredHitchcockPresents`.

**Roku:** yes, as `introSkip`.

---

### `crop` / `cropBySeason`

**Type:** `{ x, y, w, h }` of fractions `0..1` · Optional.

Trims a picture that was ripped inside a bigger frame (black surround,
watermark panel). `crop` applies to every episode; `cropBySeason` keys the same
rect by season number and wins where it has an entry:

```js
// resolved as: cropBySeason?.[season] ?? crop ?? null
cropBySeason: {
  1: { x: 0.1499, y: 0.0354, w: 0.7084, h: 0.9229 },
  2: { x: 0.1522, y: 0.0208, w: 0.7178, h: 0.9563 },
},
```

* `x`/`y` = top-left of the true picture as a fraction of the full frame;
  `w`/`h` = its size as a fraction. All four keys are required.
* Seasons with no entry (and no `crop`) are left uncropped.
* Measure per season — separate archive items are often framed differently.
  Grab a frame with `ffmpeg`, measure the picture rectangle in pixels, divide
  by the frame dimensions, and keep four decimals.
* The crop is applied to the on-glass video *and* the ambilight twin, and the
  flat-mode letterbox is sized from the cropped aspect ratio.

**Roku:** yes — resolved to a per-episode `crop` rect at build time.

---

### `incompleteEpisodes`

**Type:** `string[]` of episode **codes** · Optional.

Flags known-truncated rips in the 3D episode list so a 12-minute "episode"
doesn't read as a playback bug.

```js
incompleteEpisodes: ['S02E05', 'S02E06', 'S02E13', 'S02E14', 'S02E21', 'S02E22'],
```

Matched against `title.split(' - ')[1]` — the code, **not** the episode key and
not the index.

**Roku:** yes, as a per-episode `incomplete: true`.

---

### Quick chooser

| Situation | `grouping` | `segments` | Extra |
|---|---|---|---|
| Ordinary series, one item | `season` | `none` | — |
| Series split across several items | `season` | `none` | per-row item ids |
| Series with a/b cartoon halves | `season` | `none` | `mergeParts` + full `durations` |
| Series with an unaired pilot | `season` | `none` | `S00E00` + `seasonNames` |
| Rip with baked-in intro | `season` | `none` | `introSkipSeconds` |
| Rip framed inside a watermark | `season` | `none` | `crop` / `cropBySeason` |
| Some episodes truncated | `season` | `none` | `incompleteEpisodes` |
| TV movie (1+ parts) | `flat` | `none` | — |
| Recorded block, known lineup per tape | `flat` | `static` | `shows` (+ `segmentTimestamps`) |
| Recorded block, lineup by air date | `flat` | `lineup` | `lineups`, dated titles |
| Hosted-movie show / anime tapes | `flat` | `film` | `film` (+ `shows` for multi-film tapes) |

## Step 4 — Poster art

* Path `art/<ShowId>.<ext>`, and `data.js`'s `art` must match exactly.
* Prefer **`.jpg` or `.png`**. `.svg`/`.webp`/`.gif` work on the web but need a
  PNG sidecar for Roku (see Step 8).
* Landscape, roughly 4:3. Recent additions are `360×270`; the guide card box is
  `220×165` with `object-fit: contain`, so anything in that ballpark is fine.
* Prefer a **title card grabbed from an episode** over the archive item's
  thumbnail — item thumbnails are arbitrary frame grabs.

```bash
# grab a candidate title card ~30s in
ffmpeg -v error -ss 30 -i "https://archive.org/download/<ITEM>/<FILE>.mp4" \
  -frames:v 1 -vf scale=360:-1 -y art/<ShowId>.jpg
```

## Step 5 — `captions.js`

Required file. If you have no captions:

```js
window.CAPTIONS = {};
```

If you do, `window.CAPTIONS` maps **episode key → WebVTT text**:

```js
window.CAPTIONS = {
  "item-id::Path/To/File.mp4": "WEBVTT\n\n00:00:01.000 --> 00:00:03.000\nHello\n",
};
```

Alternative for large sets: drop files at
`shows/<ShowId>/captions/<urlEncodedEpisodeKey>.vtt` and leave `captions.js`
empty — the player falls back to that path (needs an HTTP server; `file://`
blocks track loads). `SNICK` uses the inline form and ships a 10 MB
`captions.js`; that is the only show with real captions today.

The CC badge only appears once a track actually loads, so an empty map costs
nothing.

## Step 7 — The `index.html` card

Add exactly one line inside the right genre `<div class="row">`:

```html
  <a class="card" href="engine/viewer.html?show=Newhart"><span class="art"><img src="art/Newhart.jpg" alt=""></span><span class="name">Newhart</span><span class="count">184 episodes · 8 seasons</span></a>
```

**The markup shape is parsed by a regex** in `build_catalog.py`
(`class="card"` → `href` → `<span class="art"><img src="art/…">` →
`<span class="name">` → `<span class="count">`). Copy an existing line and edit
the values; do not reorder attributes, rename classes, or insert other elements.
Whitespace and line breaks between the spans are tolerated.

Genre sections, in page order — the `<h2>` text becomes the Roku genre name:

`Animation` · `Sitcoms` · `Drama & Adventure` · `Horror & Anthology` ·
`Sketch Comedy & Late Night` · `Kids & Educational` · `TV Movies` ·
`Broadcast Blocks`

Place the card so the row stays alphabetical by display name. Adding a new
`<h2>` genre is allowed and flows through to Roku automatically.

`count` string conventions (free text, shown on the card and in the Roku guide):

| Case | Format |
|---|---|
| Multi-season series | `184 episodes · 8 seasons` |
| Single-season series | `18 episodes` |
| Partial series | `22 episodes · Season 4 only` |
| Series + extra | `249 episodes · 5 seasons + movie` |
| Merged parts | `75 parts · 2 seasons` |
| TV movie | `1 part · ~3h 7m` — add `, WOC` when the rip is with-commercials |
| Broadcast block | `8 recordings` |
| Dead source | append ` · ⚠ source currently unavailable` |

Note `show.html` derives "N episodes"/"N parts" from `grouping` on its own; this
string is only for the guide card and the Roku catalog.

## Failure modes and how they show up

| Symptom | Cause |
|---|---|
| Show missing from the Resume row | `storageProgressKey` ≠ `<showid>-progress` |
| Show missing on Roku, fine on web | `build_catalog.py` skipped it — unsupported JS in `data.js`, or the `index.html` card doesn't match the card regex |
| Episodes silently skip forward | wrong `fileHint` (404) — the player retries, then advances |
| Video plays, no audio | `.mkv`/`.avi` original with AC3/DTS and no `.mp4` derivative to fall back to |
| Everything lands in Season 1 | episode code has no `S\d+`, or the title isn't `SHOW - CODE - Name` |
| No runtimes in the list | `durations` keys don't match the episode keys (`item::file` exactly) |
| Merged episode's scrubber jumps | `mergeParts` without complete `durations` |
| Captions never appear | `CAPTIONS` key isn't the episode key, or the `.vtt` filename isn't URL-encoded |
| `ch` popup names the block instead of a show | `segments: "lineup"` with a title carrying no year at all, or one predating every `lineups` row |
| `ch` popup / FF throws on a block | missing `shows` map under `"static"`, or missing `film` map under `"film"` |
| Poster blank on Roku only | `.svg`/`.webp` poster with no PNG sidecar — run `make_roku_art.py` |
| Duplicate or half-length guide rows | alternate cuts / bumper files not excluded in Step 1b |

## Appendix: validator script

Save as `validate_show.py` in the repo root (it is not committed) and run
`python3 validate_show.py <ShowId>`. It checks the wiring that produces silent
failures: storage keys, episode-key consistency, required companion maps,
missing files, and the `index.html` card.

```python
#!/usr/bin/env python3
"""Validate one show's wiring. Usage: python3 validate_show.py <ShowId> [...]"""
import os
import re
import sys

sys.path.insert(0, os.path.expanduser("~/Projects/VaultVisionRoku/tools"))
from jsdata import JsParseError, parse_show


def check(show_id, html):
    errs, warns = [], []
    p = "shows/%s/data.js" % show_id
    if not os.path.exists(p):
        return ["%s missing" % p], []
    try:
        d = parse_show(open(p, encoding="utf-8").read())
    except JsParseError as e:
        return ["%s: %s" % (p, e)], []

    for f in ("title", "art", "storageProgressKey", "storageSettingsKey",
              "grouping", "segments", "episodes"):
        if f not in d:
            errs.append("data.js: missing required field %r" % f)
    if d.get("id", show_id) != show_id:
        errs.append("data.js id=%r != folder name %r" % (d.get("id"), show_id))
    if d.get("storageProgressKey") != show_id.lower() + "-progress":
        errs.append("storageProgressKey must be %r (Resume row depends on it)"
                    % (show_id.lower() + "-progress"))
    if d.get("storageSettingsKey") != show_id.lower() + "-settings":
        errs.append("storageSettingsKey must be %r" % (show_id.lower() + "-settings"))
    if d.get("grouping") not in ("season", "flat"):
        errs.append("grouping must be 'season' or 'flat'")
    if d.get("segments") not in ("none", "static", "film", "lineup"):
        errs.append("segments must be 'none' | 'static' | 'film' | 'lineup'")

    art = d.get("art", "")
    if art and not os.path.exists("art/" + art):
        errs.append("art/%s does not exist" % art)

    eps = d.get("episodes") or []
    if not eps:
        errs.append("no episodes")
    keys, seen = [], set()
    for i, e in enumerate(eps):
        if not isinstance(e, list) or not (2 <= len(e) <= 3):
            errs.append("episode %d: want [item, title] or [item, title, file]" % i)
            continue
        title = e[1]
        key = "%s::%s" % (e[0], e[2]) if len(e) > 2 else e[0]
        keys.append(key)
        if key in seen:
            errs.append("episode %d: duplicate key %s" % (i, key))
        seen.add(key)
        bits = title.split(" - ")
        if d.get("grouping") == "season":
            if len(bits) < 2:
                errs.append("episode %d: title %r must be 'SHOW - S01E01 - Name'"
                            % (i, title))
            elif not re.match(r"^S\d+E\d+", bits[1], re.I):
                warns.append("episode %d: code %r is not SxxEyy -> treated as season 1"
                             % (i, bits[1]))
        elif d.get("segments") == "lineup" and not re.search(r"\d\d/\d\d/\d{4}", title):
            if re.search(r"\b\d{4}\b", title):
                warns.append("episode %d: lineup title %r has only a year -> read as Jan 1"
                             % (i, title))
            else:
                errs.append("episode %d: lineup title %r has no date at all -> falls back "
                            "to the show name" % (i, title))

    dur = d.get("durations") or {}
    for k in dur:
        if k not in seen:
            warns.append("durations key matches no episode: %s" % k)
    if dur and [k for k in keys if k not in dur]:
        warns.append("%d episode(s) have no duration entry"
                     % len([k for k in keys if k not in dur]))
    if d.get("mergeParts") and not dur:
        errs.append("mergeParts needs durations to build the merged timeline")

    if d.get("segments") == "static" and not d.get("shows"):
        errs.append("segments:'static' requires a `shows` map")
    if d.get("segments") == "film" and not d.get("film"):
        errs.append("segments:'film' requires a `film` map")
    if d.get("segments") == "lineup" and not d.get("lineups"):
        errs.append("segments:'lineup' requires a `lineups` table")
    for m in ((d.get("shows") or {}), (d.get("film") or {}),
              (d.get("segmentTimestamps") or {})):
        for kk in m:
            if kk not in seen:
                errs.append("keyed map entry matches no episode: %s" % kk)

    for name, rect in (d.get("cropBySeason") or {}).items():
        if not all(x in rect for x in "xywh"):
            errs.append("cropBySeason[%s] needs x, y, w, h" % name)
    if d.get("crop") and not all(x in d["crop"] for x in "xywh"):
        errs.append("crop needs x, y, w, h")
    for code in d.get("incompleteEpisodes") or []:
        if not any(e[1].split(" - ")[1:2] == [code] for e in eps if len(e) > 1):
            errs.append("incompleteEpisodes: no episode has code %s" % code)

    for f in ("captions.js", "CREDITS.md"):
        if not os.path.exists("shows/%s/%s" % (show_id, f)):
            errs.append("shows/%s/%s missing" % (show_id, f))
    if 'viewer.html?show=%s"' % show_id not in html:
        errs.append("no card in index.html for %s" % show_id)
    return errs, warns


def main():
    html = open("index.html", encoding="utf-8").read()
    bad = 0
    for show_id in sys.argv[1:]:
        errs, warns = check(show_id, html)
        print("%s: %s" % (show_id, "OK" if not errs else "%d problem(s)" % len(errs)))
        for e in errs:
            print("  ERROR", e)
        for w in warns:
            print("  warn ", w)
        bad += len(errs)
    return 1 if bad else 0


if __name__ == "__main__":
    sys.exit(main())
```
